## PASSO 2 — ROUTER `server/routers/warehouse.ts`

Criar o router com os seguintes endpoints:

**Depósitos:** `listDepots`, `createDepot`

**Itens:** `listItems`, `createItem`, `updateItem`

**Estoque:** `checkStock` (usado pelo módulo de compras — verifica disponibilidade antes de criar SC)

**Entrada:** `registerEntry` (só após atesto de recebimento + NF)
- Atualiza `quantidadeAtual`
- Registra em `warehouseMovements`

**Saída:** `registerExit`
- Valida estoque disponível
- Atualiza `quantidadeAtual`
- Registra em `warehouseMovements`

**Empréstimo (comodato):** `registerLoan`
- Busca funcionário pelo código JFCxxxx em `employees.codigoInterno`
- Desconta do estoque
- Registra em `warehouseLoans`

**Devolução:** `returnLoanById`
- Marca como devolvido
- Devolve ao estoque

**Perdido:** `markLoanLost`
- Marca como perdido
- Cria advertência automática em `warnings`

**Fechamento do dia:** `listTodayLoans`

**Transferências:** `registerTransfer` (central↔obra, obra↔obra)

**Inventário:** `getInventorySession`, `startInventorySession`, `confirmInventoryItem`

**Equipamentos alugados:** `listRentals`, `createRental`, `requestReturn`, `confirmReturn`

**Dashboard:** `getDashboard`
- Total itens, itens críticos, valor total do estoque, empréstimos hoje, pendentes de devolução
- Lista dos 5 itens mais críticos (estoque abaixo do mínimo)
- Equipamentos alugados vencendo

```typescript
try {
  const empLoans = await db.select().from(warehouseLoans)
```

```tsx
import { useState, useEffect } from 'react';
import { Download, X } from 'lucide-react';
export function PwaInstallBanner() {
  const [prompt, setPrompt] = useState<any>(null);
  const [dismissed, setDismissed] = useState(false);
  useEffect(() => {
    const handler = (e: any) => { e.preventDefault(); setPrompt(e); };
    window.addEventListener('beforeinstallprompt', handler);
    return () => window.removeEventListener('beforeinstallprompt', handler);
  }, []);
  if (!prompt || dismissed) return null;
  const install = async () => { prompt.prompt(); const { outcome } = await prompt.userChoice; if (outcome === 'accepted') setDismissed(true); };
  return (
    <div className="fixed bottom-4 left-4 right-4 z-50 bg-[#0A192F] text-white rounded-xl p-4 shadow-2xl flex items-center gap-3 md:max-w-sm md:left-auto md:right-4">
      <div className="flex-1">
        <p className="font-bold text-sm">📱 Instalar no celular</p>
        <p className="text-xs text-gray-300 mt-0.5">Acesse mais rápido pela tela inicial</p>
      </div>
      <button onClick={install} className="bg-blue-500 text-white text-sm font-bold px-4 py-2 rounded-lg flex items-center gap-1"><Download size={14}/> Instalar</button>
      <button onClick={() => setDismissed(true)} className="text-gray-400 hover:text-white p-1"><X size={18}/></button>
    </div>
  );
}
```

## REGRAS DE NEGÓCIO — COMPRAS

### Solicitação de Compra:
- Solicitante vê apenas obras vinculadas ao seu usuário
- Itens selecionados do catálogo (não digitados livremente)
- Sistema verifica almoxarifado antes de liberar SC (usar `warehouse.checkStock`)
  - Estoque suficiente → bloqueia, obriga retirada
  - Estoque parcial → pede só a diferença
  - Sem estoque → libera normalmente
- Toggle ⚡ EMERGENCIAL → justificativa obrigatória → vai ao topo de todas as filas
- Atualizar `emergency_metrics` ao criar cada SC

### Aprovação:
- Consultar `purchase_approval_rules` para saber quem aprova
- Dentro da meta → nível 1
- Acima da meta (R$0,01) → nível 2 + alerta visual
- Abaixo de `limiteCompraDireta` → compra direta sem cotação
- SC emergencial: badge ⚡, notificação imediata, SLA configurável

### Cotação:
- Mínimo 3 fornecedores (configurável)
- Exibir histórico de preços de cada fornecedor para aquele item (de `supplier_price_history`)
- Exibir média geral e avaliação do fornecedor
- Verificar `supplier_contracts` ativos — se existir contrato vigente, pular cotação → ir direto para OC
- E-mail automático para fornecedor ao abrir cotação (usar `smtpService`)
- Ranking por `valorTotalComFrete` (item + frete)
- Verificar validade da cotação antes de gerar OC

### Ordem de Compra:
- Preencher campos fiscais automaticamente dos dados da empresa (`companies`)
- Gerar `accounts_payable` automaticamente com dados bancários do fornecedor
- Gerar `financial_events` (tipo `saida_pagamento`) automaticamente
- Se tipo = `locacao` → criar registro em `warehouse_rentals` ao receber
- Se OC de equipamento recebida → criar item em `equipment` automaticamente
- Alerta automático se prazo vencer sem recebimento

### Recebimento:
- Foto do material (usar `storagePut`)
- Foto/upload da NF
- Recebimento parcial → OC atualizada, alerta para todos, `accounts_payable` liberado proporcionalmente
- Ao concluir recebimento total → solicitar avaliação do fornecedor (opcional)
- Salvar preço em `supplier_price_history` automaticamente
- Se recebimento parcial → criar registro em `warehouse_rentals` se for locação

### Cancelamento (rollback automático):
- SC `aguardando` → qualquer um cancela → libera reserva almoxarifado
- SC `aprovada` → só ADM → + motivo obrigatório
- OC `emitida` → ADM + Diretor → cancela `accounts_payable` + libera saldo meta + reabre SC
- Registrar em `purchase_cancellations`

### Realocação de saldo:
- Só com economia confirmada (OC recebida, valor abaixo da meta)
- Saldo = meta do item - total das OCs recebidas daquele item
- Um saldo só pode ser usado uma vez — bloquear reuso
- Irreversível, rastreabilidade total

### Comissão do comprador:
- Calculada só no encerramento da obra
- Fórmula: `(meta total - gasto total) × 20%`
- Obra estourou → comissão zero
- Mais compradores → divide proporcional ao valor de OCs de cada um
- Só Diretor aprova → gera lançamento em `accounts_payable` para o comprador
